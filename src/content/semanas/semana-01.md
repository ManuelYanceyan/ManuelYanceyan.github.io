## Resumen

Durante la primera semana se realizó la exposición del sílabo del curso y la aplicación de una prueba de diagnóstico con la finalidad de identificar los conocimientos previos de los estudiantes. Se introdujeron los fundamentos del desarrollo web y el enfoque general del curso, resaltando la importancia del portafolio educativo como evidencia del aprendizaje progresivo.

![Semana 01](/imagenes/semana1.png)


## Clase
- Exposición del sílabo del curso, objetivos y alcance de la asignatura.  
  Se explicó detalladamente el contenido del sílabo, los objetivos generales y específicos del curso, así como las competencias que se esperan desarrollar a lo largo del semestre. Se aclaró el alcance de la asignatura dentro del plan de estudios y su relación con cursos previos y posteriores.

- Metodología de trabajo del curso.  
  Se presentó la metodología basada en sesiones teóricas y prácticas, trabajo colaborativo, desarrollo de laboratorios y resolución de problemas. Se enfatizó la importancia de la participación activa del estudiante y el aprendizaje autónomo.

- Sistema de evaluación y evidencias de aprendizaje.  
  Se describieron los criterios de evaluación, incluyendo prácticas calificadas, evaluaciones de logro y el portafolio educativo. Se explicó cómo cada actividad contribuye al promedio final y la importancia de cumplir con los plazos establecidos.

- Fundamentos de las tecnologías web.  
  Se introdujeron los conceptos básicos del desarrollo web, explicando qué son las tecnologías web y cómo estas permiten la creación de soluciones informáticas accesibles desde un navegador en distintos contextos organizacionales.

- Soluciones web y clasificación.  
  Se explicó de manera detallada la diferencia entre sistema web, aplicación web, sitio web y página web, analizando sus características, funciones y ejemplos de uso en el entorno real.

- Tecnologías web básicas utilizadas en el desarrollo frontend.  
  Se revisaron las principales tecnologías web, incluyendo los lenguajes de marcado para estructurar el contenido, las hojas de estilo para el diseño visual, los scripts para la interactividad y el uso de recursos gráficos y multimedia.

- Funcionamiento general de la web.  
  Se explicó el proceso completo de comunicación entre cliente y servidor, detallando la resolución de nombres de dominio mediante DNS, la transmisión de datos utilizando el protocolo TCP/IP y la transferencia de información a través del protocolo HTTP.

- Arquitectura cliente–servidor.  
  Se introdujo el concepto de arquitectura cliente–servidor, explicando el rol del navegador como cliente, el servidor web como proveedor de servicios y la interacción entre ambos mediante peticiones y respuestas.

- Roles en el desarrollo de aplicaciones para Internet.  
  Se analizaron los principales roles involucrados en el desarrollo web, describiendo las responsabilidades del desarrollador frontend en la interfaz de usuario, del desarrollador backend en la lógica del servidor y bases de datos, y del desarrollador fullstack como integrador de ambos enfoques.

- Uso de Visual Studio Code como herramienta de desarrollo.  
  Se presentó Visual Studio Code como el entorno principal de programación, explicando su interfaz, principales funcionalidades, extensiones recomendadas y la importancia de su uso para mantener un código ordenado y eficiente.

## Laboratorio
- Laboratorio 01A: Configuración de Visual Studio Code, instalación de extensiones útiles y uso de atajos de teclado.  
  Desarrollo de una página HTML básica aplicando la estructura correcta del documento y publicación del proyecto en un repositorio de GitHub como primera evidencia del portafolio educativo.
